## Bug reports

Use the GitHub bug report form and provide:

- MailVault version;
- Python version;
- operating system;
- provider profile;
- sanitized command line with secrets removed;
- relevant structured log event;
- synthetic reproduction when parser-related.

Never upload real EML, archive databases, credentials, personal data, quotations, bank details or confidential attachments.

## Feature requests

Feature requests should identify the problem, proposed scope, provider or standard involved, evidence-preservation impact, compatibility impact and alternatives considered.

## Security

Security vulnerabilities must follow [SECURITY.md](SECURITY.md), not the public issue tracker.
