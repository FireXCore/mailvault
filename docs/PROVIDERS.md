# Provider support

## Gmail IMAP

The Gmail profile uses:

- `X-GM-MSGID` as provider message identity;
- `X-GM-THRID` as provider thread identity;
- `X-GM-LABELS` as label evidence;
- `X-GM-RAW` for provider-native search;
- the mailbox advertised with `\All` as the default archive root.

Authentication uses an email address and Gmail App Password. The secret is requested through hidden input.

## Generic IMAP

The generic profile uses:

- mailbox name;
- UIDVALIDITY;
- UID;
- server internal date;
- RFC Message-ID and threading headers;
- raw EML SHA-256;
- OBJECTID `EMAILID` and `THREADID` when advertised.

The canonical archive ID is generated by MailVault and remains independent from filenames and display subjects.

## Capability negotiation

MailVault records and reacts to advertised capabilities including:

- IMAP4rev1 and IMAP4rev2;
- SPECIAL-USE;
- UIDPLUS;
- CONDSTORE;
- QRESYNC;
- OBJECTID;
- UIDONLY;
- MESSAGELIMIT;
- LITERAL+;
- IDLE.

A capability being detected does not imply that every optional optimization is enabled. The selected execution path remains conservative and evidence-preserving.

## Microsoft 365

Basic password authentication is not a portable long-term design for Exchange Online. Microsoft 365 support belongs in a separate OAuth-capable adapter and is not represented as generic password support.

## JMAP

JMAP support is planned as a separate protocol adapter. It will map JMAP Email, Thread, Blob and Mailbox identifiers into the same canonical archive model.
